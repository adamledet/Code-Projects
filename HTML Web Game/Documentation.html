Our game is a 2D shooter where multicolored shapes will fall from a random position onto the player. The player has 3 lives that are lost upon collision with objects.
Inspiration for the game comes from the classic arcade titles "Asteroids" and "Space Invaders."

The player moves along the x-axis in one dimension using the arrow keys and can fire using the spacebar.
Destroying an object awards 2 points while simply dodging one awards 1 point.
Losing all three lives exits the Update loop, causing the game to end by preventing the player from taking action. To signal this, the window turns black.

5 objects are created on startup, and there will only ever be 5 objects in the game. Rather than actually destroyed, objects that need to be removed are instead given new coordinates and attributes and reset to above the window.

The main setup for the game is a window id inside of a div created in our html code.

By referencing and appending divs to this window, we were able to create the player and objects.

Adam Ledet) Adam gave our player and objects movement. Additionally, he was completely responsible for object creation and resetting of their position.
Adam handled collision for player and asteroid using AABB collision detection.
A simpler form of detection was used for objects bouncing off of walls and to stop player movement with walls.
Adam created an array to be used to give visibility and variety to the objects. Finally, he created the lives and score for the player and implemented the Game Over screen.

Enan Munzar) Enan created the very framework for the game. He built the first window and made the player appear on the screen for the first time.
Following this, Enan made it possible to check when keys were pressed so that movement and shooting could be done with user-input.
All shooting and the delay between firing was done by Enan.
Finally, Enan created the player sprite and allowed the objects to be created with varying degrees of roundness.
